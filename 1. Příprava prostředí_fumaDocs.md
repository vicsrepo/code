Níže naleznete podrobný návod, jak nainstalovat a implementovat Fumadocs ve vašem projektu Next.js a zároveň synchronizovat dokumentaci s GitHubem.

---

## 1. Příprava prostředí

### Požadavky

- **Node.js** – verze 18 nebo novější (doporučeno: LTS)

- **Next.js** – doporučujeme používat verzi s podporou App Router

- **Správce balíčků** – npm, pnpm, yarn nebo bun

- **GitHub účet** – pro synchronizaci a správu zdrojového kódu

---

## 2. Instalace Fumadocs

Existují dvě hlavní metody, jak začít s Fumadocs:

### Automatická instalace pomocí CLI

1. Spusťte příkaz, který vytvoří novou aplikaci založenou na Fumadocs:
   
   bash
   
   ZkopírovatUpravit
   
   `npm create fumadocs-app`

2. Podle terminálových promptů vyberte:
   
   - **Název projektu** (např. `my-docs-app`)
   
   - **Content source** – doporučujeme zvolit **Fumadocs MDX** pro práci s lokálními soubory s dokumentací
   
   - **Styling** – výběr Tailwind CSS nebo jiného řešení (Tailwind je integrovaný v oficiální šabloně)

3. Po instalaci přejděte do adresáře projektu:
   
   bash
   
   ZkopírovatUpravit
   
   `cd my-docs-app`

4. Spusťte vývojový server:
   
   bash
   
   ZkopírovatUpravit
   
   `npm run dev`

*Více informací najdete v Quick Start guide.*

### Manuální instalace

Pokud chcete Fumadocs integrovat do již existujícího Next.js projektu, postupujte následovně:

1. Nainstalujte potřebné balíčky:
   
   bash
   
   ZkopírovatUpravit
   
   `npm install fumadocs-ui fumadocs-core fumadocs-mdx`

2. Nakonfigurujte MDX – přidejte vlastní soubor konfigurace (např. `source.config.ts`) pro definici kolekcí dokumentů a jejich schémat.

3. Upravte Next.js konfiguraci, aby podporovala MDX pomocí Fumadocs:
   
   js
   
   ZkopírovatUpravit
   
   `// next.config.mjs import createMDX from 'fumadocs-mdx/next'; const withMDX = createMDX(); const nextConfig = { reactStrictMode: true }; export default withMDX(nextConfig);`

*Podrobný manuál instalace naleznete v Manual Installation guide.*

---

## 3. Konfigurace zdroje obsahu

Pro správu dokumentů se doporučuje použít **Fumadocs MDX** jako zdroj obsahu:

1. **Definice kolekce**  
   Vytvořte soubor `source.config.ts` a definujte kolekci dokumentů:
   
   ts
   
   ZkopírovatUpravit
   
   `import { defineDocs } from 'fumadocs-mdx/config'; export const docs = defineDocs({  dir: 'content/docs',  docs: { /* volitelné nastavení kolekce */ },  meta: { /* volitelné nastavení metadat */ }, });`

2. **Načtení zdroje**  
   V souboru (např. `lib/source.ts`) načtěte generovaná data a převeďte je pomocí funkce `loader`:
   
   ts
   
   ZkopírovatUpravit
   
   `import { docs } from '@/.source'; import { loader } from 'fumadocs-core/source'; export const source = loader({  baseUrl: '/docs',  source: docs.toFumadocsSource(), });`

## 4. Vytvoření layoutů a stránek

### Hlavní layout aplikace

Zajistěte, aby byl celý obsah zabalen do provideru. Například:

tsx

ZkopírovatUpravit

`// app/layout.tsx import { RootProvider } from 'fumadocs-ui/provider'; import type { ReactNode } from 'react';  export default function Layout({ children }: { children: ReactNode }) {  return (    <html lang="cs">      <body style={{ display: 'flex', flexDirection: 'column', minHeight: '100vh' }}>        <RootProvider>{children}</RootProvider>      </body>    </html>   ); }`

### Layout pro dokumentaci

V adresáři `/app/docs` vytvořte soubor `layout.tsx`, který využije komponentu pro dokumentaci:

tsx

ZkopírovatUpravit

`// app/docs/layout.tsx import { DocsLayout } from 'fumadocs-ui/layouts/docs'; import { baseOptions } from '@/app/layout.config'; import { source } from '@/lib/source'; import type { ReactNode } from 'react';  export default function DocsLayoutWrapper({ children }: { children: ReactNode }) {  return (    <DocsLayout tree={source.pageTree} {...baseOptions}>       {children}    </DocsLayout>   ); }`

### Dynamické generování stránek

Pro zachycení všech dokumentů vytvořte dynamickou routu:

tsx

ZkopírovatUpravit

`// app/docs/[[...slug]]/page.tsx import { DocsPage, DocsBody, DocsTitle, DocsDescription } from 'fumadocs-ui/page'; import { notFound } from 'next/navigation'; import defaultMdxComponents from 'fumadocs-ui/mdx'; import { source } from '@/lib/source';  export default async function Page({ params }: { params: { slug?: string[] } }) {  const page = source.getPage(params.slug);  if (!page) notFound();  const MDX = page.data.body;  return (    <DocsPage toc={page.data.toc} full={page.data.full}>      <DocsTitle>{page.data.title}</DocsTitle>      <DocsDescription>{page.data.description}</DocsDescription>      <DocsBody>        <MDX components={{ ...defaultMdxComponents }} />      </DocsBody>    </DocsPage>   ); }  export async function generateStaticParams() {  return source.generateParams(); }`

## 5. Synchronizace s GitHub

Fumadocs umožňuje synchronizovat obsah dokumentace s GitHub repozitářem. Tento proces můžete implementovat následovně:

### Možnosti synchronizace:

1. **Remote Content Loader**  
   Fumadocs podporuje načítání souborů z GitHubu pomocí API. Doporučujeme použít řešení, kdy se při každém nasazení nebo pomocí cron jobu spustí skript, který:
   
   - Načte aktuální strukturu dokumentů (page tree) z GitHub repozitáře
   
   - Vygeneruje soubor (např. `page-tree.json`), který se poté načítá do Fumadocs
   
   - V lokálním vývojovém režimu můžete využít lokální souborový systém, zatímco v produkci se používá GitHub API pro aktualizaci

2. **Automatizace pomocí CI/CD**  
   Při každém commitu do repozitáře můžete využít GitHub Actions ke spuštění skriptu, který:
   
   - Spustí generátor dokumentů (např. pomocí příkazu `node ./scripts/generate-docs.mjs`)
   
   - Aktualizuje synchronizovaný obsah v repozitáři
   
   - Volitelně aktivuje hot-reloading v aplikaci

### Příklad konfigurace synchronizace:

*V souboru `lib/source.ts` můžete přidat funkci pro připojení souboru z GitHubu:*

ts

ZkopírovatUpravit

`import { loader } from 'fumadocs-core/source'; import { attachFile } from 'fumadocs-openapi/server'; // pokud používáte OpenAPI nebo vlastní synchronizační logiku  export const source = loader({  baseUrl: '/docs',  source: docs.toFumadocsSource(),  pageTree: {     attachFile, // dekoruje položky stránkového stromu badge či dalšími informacemi z GitHubu   }, });`

### Další kroky:

- **Ověření a testování**: Po nastavení synchronizace proveďte testy, zda se změny v GitHub repozitáři správně načítají a aplikují.

- **Dokumentace a reference**: Podrobnější informace o implementaci synchronizace najdete v [dokumentaci Fumadocs pro GitHub integraci](https://fumadocs.vercel.app/docs/headless/remote/github) a v diskusních tématech na GitHubu (např. Built-In Support for GitHub Content Loading).
